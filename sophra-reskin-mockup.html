Phase 2 — Beverage Brand Re-skin (mockup)

Visual-only pass on the Beverage app. No behavior, route, or data changes — same flows, same DB, same code paths. What changes: colors, typography, the logo, button + card styles. Goal is to make Beverage feel like the same product family as Schedule — when you click "Schedule" or "Beverage" from the launcher, the visual language should carry over.

A. Dashboard + sidebar — before vs after

Sidebar is the most visible change. Logo "M" becomes the Sophra icon. Amber accent color (current "Wine" branding) becomes Sophra olive. Stone neutrals stay similar in feel but warm up to cream/brown for cohesion with Schedule.

D. What's NOT changing

Out of scope for Phase 2 — listed explicitly so we don't drift.

Same as today • All routes (/dashboard, /dashboard/recipes, etc.) • All forms, validation, DB writes
• Prisma schema, Supabase auth, role-based access
• Page logic, server actions, API routes
• Existing icons (Lucide set)
• Sidebar layout / collapse behavior
• Cookie / SSO setup (Phase 1 left this stable)

Renames worth doing now package.json "name": "meyhouse" → "sophra-beverage"; page <title> tags; any "Meyhouse" string in user-facing copy.
